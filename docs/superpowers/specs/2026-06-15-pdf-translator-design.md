# PDF Translator - 设计规格文档

**日期:** 2026-06-15  
**项目名称:** PDF Translator  
**版本:** v1.0

---

## 1. 项目概述

### 1.1 项目目标
开发一款轻量级桌面应用程序，用于中英互译PDF文档，保持原文格式和排版。

### 1.2 核心功能
- 提取PDF中的文本内容
- 将文本翻译为目标语言（中↔英）
- 将翻译后的文本替换回原PDF，保持原有格式
- 简洁易用的图形界面

### 1.3 目标用户
偶尔需要翻译PDF文件的用户，不需要复杂的批量处理功能。

## 2. 技术方案

### 2.1 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 桌面框架 | **Tauri v2** | 轻量级、安全、跨平台 |

### 2.2 翻译API支持
支持多种第三方翻译API，用户可选择：

| API | 说明 | 费用 |
|-----|------|------|
| Google Translate | 广泛支持 | 有免费额度 |
| DeepL | 翻译质量高 | 有免费额度 |
| 百度翻译 | 中文优化 | 有免费额度 |

**注意:** 用户需自行提供API密钥。

## 3. 功能规格

### 3.1 核心功能列表

#### F1: PDF文件选择
- 支持拖拽上传PDF文件到应用窗口
- 支持点击按钮打开文件选择对话框
- 显示选中文件的基本信息（文件名、页数、大小）
- 验证文件是否为有效的PDF格式

#### F2: 翻译方向选择
- 单向切换：中文→英文 或 英文→中文
- 清晰的视觉指示当前选择的翻译方向

#### F3: API配置
- 输入框供用户输入API密钥
- 下拉选择翻译服务提供商（Google/DeepL/百度）
- "保存配置"按钮将设置保存到本地
- 下次启动时自动加载已保存的配置
- API密钥安全存储（使用系统密钥链或加密存储）

#### F4: 翻译执行
- "开始翻译"主按钮
- 进度条显示翻译进度（已处理页数/总页数）
- 支持取消操作
- 错误处理：网络错误、API限流、PDF解析失败等

#### F5: 结果保存
- 翻译完成后自动打开文件保存对话框
- 默认文件名：原文件名 + "_translated" 后缀
- 保存成功后显示成功提示

```

│  密钥: [••••••••••••••]                                     │

└─────────────────────────────────────────────────────────────┘
```

### 3.3 状态定义

| 状态 | 说明 | UI表现 |
|------|------|--------|
| IDLE | 初始状态，等待选择文件 | 翻译按钮禁用 |
| READY | 已选择文件，可开始翻译 | 翻译按钮启用 |
| TRANSLATING | 翻译中 | 显示进度条，按钮变为"取消" |
| SUCCESS | 翻译完成 | 显示成功提示，启用重新翻译 |
| ERROR | 发生错误 | 显示错误信息，提供重试选项 |

## 4. UI/UX 设计

### 4.1 设计风格
- **风格:** 简洁、现代、轻量
- **主题:** 支持深色/浅色模式切换（跟随系统）
- **色调:** 中性灰为主色，蓝色为强调色

### 4.2 布局结构
- **单窗口设计:** 所有功能在一个窗口内完成
- **居中布局:** 内容居中，最大宽度限制
- **响应式:** 窗口可调整大小，布局自适应

### 4.3 组件清单

| 组件 | 描述 | 状态 |
|------|------|------|
| DropZone | 文件拖放区域 | default, hover, dragging, hasFile |
| FileInfo | 显示文件信息 | - |
| DirectionToggle | 翻译方向切换 | zh-en, en-zh |
| ApiConfig | API配置表单 | collapsed, expanded |
| TranslateButton | 主操作按钮 | idle, loading, disabled |
| ProgressBar | 翻译进度条 | - |
| Toast | 操作反馈提示 | success, error, info |

### 4.4 交互反馈
- 按钮悬停: 轻微放大 + 颜色变化
- 文件拖入: 边框高亮 + 背景色变化
- 翻译进度: 实时百分比 + 页数显示
- 操作成功: Toast提示，3秒后自动消失

### 5.2 核心模块

#### Frontend (React)

| 模块 | 职责 |
|------|------|
| `DropZone` | 处理文件拖放和选择 |
| `useTranslation` | 管理翻译状态和流程 |
| `services/pdf.ts` | 调用Rust端的PDF处理功能 |
| `services/translator.ts` | 调用翻译API |

#### Backend (Rust/Tauri)

| 模块 | 职责 |
|------|------|
| `pdf_commands` | Tauri命令：读取PDF文本、写入翻译文本 |
| `config_store` | 安全存储API配置 |

### 5.3 数据流

```
[PDF文件] 
    ↓
[Rust: 提取文本块]
    ↓
[React: 分批发送到翻译API]
    ↓
[Rust: 按原位置替换文本]
    ↓
[新PDF文件]
```

## 6. 错误处理

### 6.1 错误类型

| 错误类型 | 原因 | 处理方式 |
|----------|------|----------|
| INVALID_FILE | 非PDF文件 | 提示用户选择有效的PDF文件 |
| PDF_PARSE_ERROR | PDF解析失败 | 提示用户文件可能损坏 |
| NETWORK_ERROR | 网络连接问题 | 显示错误，提供重试 |
| API_ERROR | API返回错误 | 显示API错误信息 |
| RATE_LIMIT | API限流 | 提示稍后再试 |

### 6.2 用户提示
- 所有错误都显示友好的中文提示
- 提供明确的解决建议
- 关键操作失败后可重试

## 7. 非功能需求

### 7.1 性能
- 应用启动时间: < 2秒
- 10页PDF翻译: < 30秒（网络正常情况下）
- 内存占用: < 200MB

### 7.2 兼容性
- Windows 10/11
- 安装包体积: < 20MB

### 7.3 安全性
- API密钥加密存储
- 不上传用户文件到第三方服务器（仅文本内容）
